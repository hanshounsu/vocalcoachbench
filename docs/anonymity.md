# Anonymous Release Checklist

This repository is prepared for anonymous peer review.

Before linking it in a submission system:

- Use a fresh repository with no identifying commit history.
- Keep commit authors anonymous.
- Do not include personal names, email addresses, institutions, lab names, local
  filesystem paths, or private server paths.
- Host dataset files separately and ensure the dataset metadata is also
  anonymized for review.
- Prefer an anonymous repository proxy when the review venue requires
  double-blind submission.

This repository contains evaluator/scorer code, prompt templates, format
documentation, and toy examples only. Benchmark audio files and full annotation
files are expected to be hosted separately.
